## Phase 2 — 零點擊資訊（Zero-click）

**目標：使用者不用打開 App 就能獲得推薦。**

### 2-1 智慧通知系統
- **關鍵提醒 vs 安靜提醒**
  - 回饋率 ≥ 5%：主動推播通知（含震動）
  - 回饋率 < 5%：安靜通知 / 僅顯示在 Widget
  - 快閃活動（限時高回饋）：強提醒 + 倒數
- **通知內容優化**
  - 一句話推薦：「星巴克刷 CUBE 卡，樂饗購方案 3.3%！」
  - 加入預估回饋金額：「這杯拿鐵幫你省 5 元」
- **頻率控制**
  - 同一商家 24 小時內只通知一次
  - 每日通知上限 5 則

### 2-2 背景 Geofencing（無 UI 觸發）
- 移除對地圖介面的依賴，改為純背景定位觸發
- 預存全台主要商家座標（便利商店、加油站、百貨等）
- 進入範圍 → 背景推播，不需開啟 App
- 技術：Service Worker + Background Sync API

### 2-3 Badging API
- PWA 主畫面 icon 右上角顯示數字
- 數字 = 附近 500m 內有高回饋的商家數量
- 使用者一眼就知道該不該打開 App

## Phase 3 — 語音與捷徑整合

**目標：連手機都不用掏出來。**

### 3-1 Siri 捷徑整合
- 提供 API endpoint：`GET /api/voice?lat={lat}&lng={lng}`
- 回傳純文字：「這家店刷國泰 CUBE 卡，選樂饗購方案，回饋 3.3%！」
- 使用者建立 iOS 捷徑：「嘿 Siri，這家店刷哪張？」
- 捷徑流程：取得目前位置 → 呼叫 API → 語音朗讀結果

### 3-2 Google Assistant 整合
- 同上 API，串接 Google Assistant Routines
- Android 使用者可語音查詢

### 3-3 Apple Watch / Wear OS
- 極簡介面：只顯示「卡名 + 回饋率」
- 抬腕即看，結帳前 2 秒決策

## Phase 4 — 消費追蹤與回饋驗證

**目標：刷完卡後自動告訴你「省了多少」，形成正向循環。**

### 4-1 電子發票載具聯動
- **方案 A — 網頁爬蟲**
  - 使用者輸入手機條碼 + 驗證碼
  - 後台定時（每小時）爬取財政部電子發票平台
  - 比對消費紀錄 vs 推薦卡片
- **方案 B — 郵件追蹤**
  - 串接 Gmail API，讀取發票通知信
  - 解析消費金額與商家
- **回饋計算**
  - 「這筆 1,000 元消費，用 A 卡成功賺到 50 元回饋！」
  - 月度報告：「本月總回饋 NT$ 1,230，比亂刷多省 NT$ 780」

### 4-2 消費儀表板
- 月度消費分類圓餅圖
- 各卡片回饋累計排行
- 「如果你用錯卡，會少賺多少」對比

### 4-3 回饋到期提醒
- 追蹤各銀行點數/回饋到期日
- 到期前 7 天提醒兌換

## Phase 5 — 平台化與社群

**目標：從個人工具變成社群平台。**

### 5-1 卡片評測與比較
- 使用者投稿「實際回饋經驗」
- 系統化的卡片 vs 卡片比較表
- 辦卡推薦（依消費習慣）

### 5-2 優惠情報聚合
- 爬取各銀行官網的限時活動
- 快閃優惠即時推播
- 使用者回報「這個優惠還有效」

### 5-3 開放 API
- 讓第三方記帳 App 串接 CardBrain 推薦引擎
- B2B：提供銀行「用戶刷卡行為分析」（去識別化）

## Phase 6 — 原生功能（需跳脫純 PWA）

**目標：突破瀏覽器限制，提供原生體驗。**

### 6-1 iOS Widget / Live Activities
- 技術：Capacitor 或 React Native 包裝原生層
- 負一屏 Widget：根據位置/時間自動顯示最優卡
- Live Activities：進入商圈時，鎖定畫面顯示推薦卡片

### 6-2 Android PWA Widget
- Android 已支援 PWA Widgets API
- 桌面 Widget 顯示「當前最優卡 + 附近優惠數」

### 6-3 NFC 觸發
- 未來手機 NFC 感應刷卡前，攔截並提示最優卡
- 需要深度原生整合（最遠期）
