## Phase 2 — 零點擊資訊（Zero-click）

**目標：使用者不用打開 App 就能獲得推薦。**

### 2-1 智慧通知系統 `Stone` `Alan`

- **分級提醒**
  - 回饋率 ≥ 5%：主動推播（含震動）
  - 回饋率 < 5%：安靜通知 / 僅 Widget
  - 快閃活動（限時高回饋）：強提醒 + 倒數
- **內容優化**
  - 一句話推薦：「星巴克刷 CUBE 卡，樂饗購方案 3.3%！」
  - 預估回饋金額：「這杯拿鐵幫你省 5 元」
- **頻率控制**
  - 同一商家 24 小時內只通知一次
  - 每日通知上限 5 則

### 2-2 背景 Geofencing `Rita` `Alan`

- 移除對地圖介面的依賴，純背景定位觸發
- 預存全台主要商家座標（便利商店、加油站、百貨等）
- 進入範圍 → 背景推播，不需開啟 App
- 技術：Service Worker + Background Sync API

### 2-3 Badging API `Stone`

- PWA 主畫面 icon 右上角顯示數字
- 數字 = 附近 500m 內高回饋商家數量

## Phase 3 — AI Agent 智慧客服（取代 Siri 捷徑）

**目標：內建對話式推薦，比語音助手更靈活。**

### 3-1 對話式推薦引擎 `Alan`

- API endpoint：`POST /api/chat`
- 串接 LLM（Claude API / OpenAI）+ 現有卡片資料庫作為 knowledge base
- 支援自然語言查詢：
  - 「我在全聯要刷哪張卡？」
  - 「下個月去日本，帶哪幾張卡最划算？」
  - 「CUBE 卡這個月還剩多少回饋額度？」
- 上下文記憶：知道使用者的卡組合，不用每次重複

### 3-2 對話 UI `Stone` `Hera`

- 聊天氣泡介面，嵌入 App 右下角
- 支援快捷問題按鈕（常見問題一鍵問）
- 推薦結果可直接點擊查看詳情

### 3-3 優勢（vs Siri 捷徑）

| | Siri 捷徑 | 內建 AI Agent |
|---|---|---|
| 設定門檻 | 使用者要自建捷徑 | 零設定，開箱即用 |
| 平台 | 僅 iOS | 跨平台（PWA） |
| 對話能力 | 一問一答 | 多輪對話 + 上下文 |
| 複雜查詢 | 不支援 | 「比較 A 卡和 B 卡去日本哪張好」 |

## Phase 4 — 消費追蹤與回饋驗證

**目標：刷完卡後自動告訴你「省了多少」，形成正向循環。**

### 4-1 電子發票載具聯動 `Alan`

- **方案 A — 網頁爬蟲**：手機條碼 + 驗證碼 → 爬取財政部平台
- **方案 B — 郵件追蹤**：Gmail API 讀取發票通知信
- 回饋計算：「這筆 1,000 元，用 A 卡成功賺到 50 元！」

### 4-2 消費儀表板 `Stone` `Hera`

- 月度消費分類圓餅圖
- 各卡片回饋累計排行
- 「如果你用錯卡，會少賺多少」對比

### 4-3 回饋到期提醒 `Alan`

- 追蹤各銀行點數/回饋到期日
- 到期前 7 天提醒兌換
