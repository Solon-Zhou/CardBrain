# 🧠 CardBrain 2.0：Zero-Click 隨身刷卡大腦

> **核心哲學：LLM 負責聽懂，引擎負責算錢，OS 負責觸發。**

---

## 一、核心痛點與破局點

| 痛點 | 描述 |
|------|------|
| **忘記開 App** | 結帳時想不起來要開 App，背景推播又太煩，被系統殺掉 |
| **通用 AI 算不準** | 信用卡規則太複雜（通路限制、每月上限），ChatGPT/Gemini 容易數學幻覺，且沒有使用者的持卡狀態 |

**破局點：「反客為主，精準攔截」+「LLM 負責聽懂，引擎負責算錢」**

---

## 二、三大主攻功能

### 🔋 戰鬥力偵測機（精算引擎）

不看虛假的「最高 10%」，系統帶入「每月回饋上限」與「真實消費金額」精算，告訴你到底能拿回多少錢。

- 輸入：商家 + 消費金額
- 輸出：實際回饋金額（非回饋率），含月上限警示
- 範例：「CUBE 卡在星巴克消費 300 元 → 實拿 9.9 元（本月已用 2,100/3,000 額度）」

### 🃏 手牌最佳化（現有卡包）

根據定位或輸入，只從使用者「已經擁有的卡片」中，挑出當下最優解。

- 輸入：地點 or 商家 + 使用者卡組合
- 輸出：最佳卡片 + 精算回饋
- 與 1.0 差異：加入月累計消費追蹤，當某張卡的月上限快到時自動推薦次優卡

### 🧭 許願池指南針（辦卡推薦）

使用者輸入目的，系統反向推薦最適合去辦的新卡。**1.0 沒有的全新功能。**

- 輸入：「下個月去日本玩預計花 5 萬」
- 輸出：推薦辦哪張卡 + 預估回饋金額 + 辦卡連結
- 範例：「建議辦富邦 J 卡，日本消費 3% 回饋，5 萬元可拿 1,500 元」

## 三、技術架構

> 放棄高成本的手機 App 開發，將火力集中在後端，前端體驗交給系統原生機制。

```
┌─────────────────────────────────────────────────┐
│                   觸發層（免開發）                 │
│  iOS 捷徑 / Android Tasker / LINE Pay 攔截       │
│  「打開 Apple Wallet 時 → 自動呼叫 API」          │
└──────────────────────┬──────────────────────────┘
                       │ POST /api/brain
                       ▼
┌─────────────────────────────────────────────────┐
│                 大腦後端（FastAPI）                │
│                                                  │
│  ┌──────────────┐    ┌────────────────────────┐  │
│  │  LLM 意圖層  │───▶│   Python 精算引擎      │  │
│  │  抽取：地點   │    │   - 商家比對           │  │
│  │  金額、目的   │    │   - 月上限追蹤         │  │
│  └──────────────┘    │   - 回饋金額精算       │  │
│                      │   - 辦卡反向推薦       │  │
│                      └───────────┬────────────┘  │
│                                  │               │
│                      ┌───────────▼────────────┐  │
│                      │   卡片 / 規則 DB       │  │
│                      │   (SQLite / Supabase)  │  │
│                      └────────────────────────┘  │
└──────────────────────┬──────────────────────────┘
                       │ 純文字回應
                       ▼
┌─────────────────────────────────────────────────┐
│                   回應層                          │
│  系統原生通知 / 捷徑結果顯示 / PWA 對話介面       │
│  📍 星巴克：推薦刷【CUBE卡】拿 3.3%，實省 9.9 元  │
└─────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────┐
│              設定前台（PWA，退居幕後）              │
│  純粹作為「勾選我有什麼卡」的管理介面              │
└─────────────────────────────────────────────────┘
```

### 極簡 API 設計

```
POST /api/brain
Content-Type: application/json

// 自然語言模式
{ "query": "我在星巴克要花 300", "card_ids": [1, 5, 12] }

// 經緯度模式
{ "lat": 25.033, "lng": 121.565, "amount": 300, "card_ids": [1, 5, 12] }

// 辦卡推薦模式
{ "query": "下個月去日本玩預計花 5 萬", "mode": "suggest_new_card" }
```

```
回應（純文字，方便捷徑直接顯示）：

📍 星巴克
💳 推薦刷【CUBE 卡】樂饗購方案
💰 回饋 3.3%，消費 300 元實拿 9.9 元
⚠️ 本月已用額度 2,100 / 3,000 元
```

## 四、與 1.0 的關係

### 可直接複用

| 模組 | 說明 |
|------|------|
| FastAPI 後端 | 加一支 `/api/brain` endpoint |
| 卡片 / 規則 DB | 19 銀行 43 卡 133 規則，直接沿用 |
| 推薦引擎 | `query.py` 的 `recommend_by_merchant` 作為精算引擎基礎 |
| 商家對照表 | `merchant_aliases.py` 沿用 |
| PWA 卡片管理 | 「我的卡組合」功能保留 |

### 需要新增

| 模組 | 負責 | 說明 |
|------|------|------|
| LLM 意圖抽取 | Alan | 解析自然語言 → `{merchant, amount, purpose, location}` |
| 月累計追蹤 | Alan | DB 新增使用者消費記錄，追蹤各卡月回饋累計 |
| 精算引擎升級 | Alan | 帶入月上限 + 真實金額，算出實際回饋金額 |
| 辦卡推薦邏輯 | Alan | 反向推薦：根據目的推薦使用者「沒有但該辦」的卡 |
| 極簡 API | Alan + Sunu | `POST /api/brain`，統一入口 |
| iOS 捷徑範本 | Sunu | 提供 `.shortcut` 一鍵匯入檔 |
| PWA 簡化 | Stone | 砍掉地圖/搜尋主頁，只留卡管理 + AI 對話 |
| 對話 UI | Stone + Hera | 聊天介面（如果要保留 PWA 內對話功能） |

### 可以不做了

| 原計畫 | 原因 |
|--------|------|
| ❌ 背景 Geofencing | 改由 iOS 捷徑 / Tasker 觸發，不靠 Service Worker |
| ❌ Siri 捷徑開發 | 已在 2.0 前砍掉 |
| ❌ 原生 App | 觸發層交給 OS 自動化，不需要原生開發 |
| ❌ Badging API | PWA 退居幕後，badge 數字無意義 |
| ❌ 地圖互動優化 | 地圖不再是主介面 |

## 五、「作弊級」觸發策略

### iOS 捷徑 Automation

```
觸發條件：打開「Apple Wallet」App 時
   → 取得目前位置（經緯度）
   → GET /api/brain?lat=...&lng=...&card_ids=...
   → 顯示通知：推薦結果
```

使用者只要匯入一次捷徑，之後每次打開錢包 App 就自動觸發推薦。

### Android Tasker

```
觸發條件：打開 LINE Pay / Google Pay 時
   → 取得目前位置
   → 呼叫 API
   → 系統通知顯示結果
```

### 優勢

- **零開發成本**：不用寫任何 native code
- **不會被系統殺**：是使用者主動設定的 Automation，OS 不會限制
- **精準時機**：使用者打開錢包 = 準備付款，這是最佳推薦時機
- **跨平台**：iOS 捷徑 + Android Tasker 覆蓋兩大平台
