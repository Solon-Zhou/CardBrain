# 🧠 CardBrain 3.0：一個引擎，三種省錢模式

> **核心哲學：LLM 負責聽懂，精算引擎負責算錢，三個場景負責讓人心痛到行動。**

---

## 一、為什麼需要 3.0？

### 2.0 的問題

2.0 解決了「怎麼觸發」（iOS 捷徑 / Tasker），但沒解決更根本的問題：

| 問題 | 說明 |
|------|------|
| **痛點不夠痛** | 「星巴克省 10 元」不會讓人裝 App |
| **沒有護城河** | 查表 + 乘法，任何人可以複製 |
| **商業模式空白** | 使用者省了錢，但我們怎麼賺？ |
| **沒有留存理由** | 查完就走，不會再開 |

### 3.0 的突破

**不只告訴你「現在刷哪張」，還告訴你「你已經虧了多少」和「怎麼規劃才省最多」。**

用一個精算引擎，驅動三種場景，形成飛輪：

```
後悔（過去）→ 心痛 → 開始用即時推薦（現在）→ 養成習慣
    → 大額規劃（未來）→ 辦新卡 → 我們賺佣金
         → 更多卡 → 更需要推薦 → 飛輪轉起來
```

## 二、精算引擎（共用核心）

所有模式共用同一個引擎，只建一次。

```
┌─────────────────────────────────────────────┐
│              精算引擎 (Calculator Engine)     │
│                                              │
│  輸入：商家 + 金額 + 使用者卡組合             │
│                                              │
│  ┌──────────────────────────────────────┐    │
│  │ 1. 商家比對（merchant_aliases）       │    │
│  │ 2. 回饋規則查詢（rewards DB）         │    │
│  │ 3. 月上限追蹤（已用額度 / 剩餘額度）   │    │
│  │ 4. 實際金額精算（不是回饋率，是錢）    │    │
│  │ 5. 多卡排序（考慮上限後的真實排名）    │    │
│  └──────────────────────────────────────┘    │
│                                              │
│  輸出：每張卡的實際回饋金額 + 排名 + 警示     │
└─────────────────────────────────────────────┘
```

### 與 1.0 引擎的差異

| | 1.0 | 3.0 |
|---|---|---|
| 輸出 | 回饋率 3.3% | **實拿 9.9 元**（帶入金額精算） |
| 月上限 | 僅顯示警示文字 | **追蹤累計消費，上限到了自動推次優卡** |
| 排序依據 | 回饋率高低 | **實際金額高低**（率高但上限到了 = 排後面） |
| 多筆消費 | 不支援 | **輸入多筆，拆解最佳多卡策略** |

## 三、三種省錢模式

### 模式 1：即時推薦（現在）

> 「我現在在這家店，刷哪張？」

**使用者：日常消費，快速決策。**

- 輸入：商家名 or 定位 + 消費金額
- 輸出：最佳卡片 + 實際回饋金額
- 管道：PWA 對話 / iOS 捷徑 / Tasker

```
使用者：「我在全聯要花 2,000」
CardBrain：
  💳 推薦【聯邦賴點卡】LINE Pay 支付
  💰 回饋 3%，實拿 60 元
  ⚠️ 本月已用 4,200 / 5,000 元上限，剩餘額度 800 元
  💡 超過 800 元的部分建議改刷【玉山 U Bear】2%
```

**價值：門檻最低，讓使用者開始用。**

### 模式 2：後悔計算機（過去）

> 「你上個月用錯卡，少拿了 $1,280。」

**使用者：輸入過去的消費紀錄，看自己虧了多少。**

- 輸入：多筆消費（手動輸入 or 未來串載具）
- 輸出：「你實拿 vs 最佳方案」的差距
- 累積數字 → 使用者越看越心痛 → 行為改變

```
使用者輸入上個月 5 筆消費：

| 商家      | 金額    | 你刷的卡   | 實拿   | 最佳卡     | 可拿   |
|-----------|---------|-----------|--------|-----------|--------|
| 全聯      | 3,000   | 台新玫瑰  | 30     | 聯邦賴點   | 90     |
| 星巴克    | 500     | 中信 LINE | 25     | CUBE 卡   | 16.5   |
| 中油      | 2,000   | 國泰世華  | 20     | 中信英雄   | 60     |
| 家樂福    | 5,000   | 現金      | 0      | 聯邦賴點   | 150    |
| momo      | 8,000   | 台新玫瑰  | 80     | 玉山 U Bear| 240    |
|           |         | **你拿到** | **155**| **最佳**  | **556.5**|

😱 你上個月少拿了 $401！一年下來約虧 $4,812。
```

**價值：製造心痛感 → 留存動力 + 分享傳播（「我居然虧了這麼多」）。**

### 模式 3：大額規劃顧問（未來）

> 「去日本花 10 萬，怎麼刷最省？」

**使用者：規劃大額消費（旅遊 / 3C / 裝潢），系統拆解最佳多卡策略。**

- 輸入：消費計畫（目的地 + 預算 + 消費類別）
- 輸出：多卡策略 + 總省金額 + 辦卡建議
- 觸發辦卡 → 聯盟行銷佣金

```
使用者：「下個月去日本 7 天，預算 10 萬」

CardBrain 規劃：

  ✈️ 機票 20,000 → 刷【匯豐旅人卡】3% = 省 600 元
  🏨 飯店 30,000 → 刷【國泰 CUBE】Booking 5% = 省 1,500 元
  🛍️ 購物 30,000 → 刷【富邦 J 卡】日本 3% = 省 900 元
  🍱 餐飲 15,000 → 刷【玉山 U Bear】海外 2.8% = 省 420 元
  🚃 交通  5,000 → 刷【富邦 J 卡】日本 3% = 省 150 元

  💰 總省：$3,570
  🆕 你沒有富邦 J 卡？→ 建議申辦（海外消費最佳）[申辦連結]
```

**價值：大額 = 省的錢有感 + 辦卡推薦 = 營收。**

## 四、商業模式

```
          使用者省錢                 CardBrain 賺錢
              │                          │
  ┌───────────▼──────────┐   ┌──────────▼───────────┐
  │ 後悔計算機            │   │                      │
  │ 「你虧了 $4,812/年」  │──▶│ 使用者開始認真用      │
  │                      │   │ （留存 + DAU）        │
  └──────────────────────┘   └──────────────────────┘
                                        │
  ┌──────────────────────┐   ┌──────────▼───────────┐
  │ 大額規劃              │   │                      │
  │ 「去日本省 $8,700」   │──▶│ 推薦辦新卡            │
  │ 「你該辦這張卡」      │   │ 聯盟行銷佣金          │
  └──────────────────────┘   │ $500~2,000 / 核卡    │
                              └──────────────────────┘
```

| 營收來源 | 說明 | 時機 |
|---------|------|------|
| **辦卡聯盟行銷** | 使用者透過推薦連結申辦新卡，銀行付佣金 | 大額規劃推薦辦卡時 |
| **B2B 數據 API** | 去識別化的消費趨勢數據，賣給銀行/金融機構 | 使用者規模夠大時 |
| **Premium 訂閱** | 進階功能：自動載具串接、月報推送、多人家庭方案 | 產品成熟後 |

**Hackathon 重點講辦卡聯盟行銷 — 最具體、最容易理解。**

### API 設計

## 六、與 1.0 / 2.0 的關係

| 模組 | 1.0 | 2.0 | 3.0 |
|------|-----|-----|-----|
| 卡片管理 PWA | 主介面 | 退居幕後 | **保留，加入三模式入口** |
| 推薦引擎 | 回饋率排序 | 加入金額精算 | **精算 + 月上限 + 多卡拆解** |
| 地圖 Geofencing | 主打功能 | 砍掉 | **可選附加（不是重點）** |
| LLM | 無 | 意圖抽取 | **意圖抽取 + 旅遊規劃拆解** |
| 後悔計算機 | 無 | 無 | **新增** |
| 大額規劃 | 無 | 辦卡推薦 | **升級為多卡拆解策略** |
| 商業模式 | 無 | 無 | **辦卡聯盟行銷** |
| 觸發方式 | 開 App | iOS 捷徑 | **PWA 對話 + 捷徑（並行）** |

## 八、Hackathon Demo 劇本（五幕）

```
第一幕 — 製造共鳴（30 秒）
  「在場有多少人有 3 張以上信用卡？」（舉手）
  「你每次結帳前，會想 10 秒鐘該刷哪張嗎？」
  「大多數人不會。然後你就默默虧了錢。」

第二幕 — 揭露痛苦：後悔計算機（60 秒）
  「我們來算算看，上個月的 5 筆消費你虧了多少。」

  「你上個月少拿了 $401。一年就是 $4,812。」
  （停頓，讓數字沉澱）

第三幕 — 解決現在：即時推薦（30 秒）
  「從現在開始，結帳前問一句就好。」
  對話框輸入：「我在全聯要花 2,000」
  秒回：聯邦賴點卡，實拿 60 元，上限剩 800 元
  「比問 ChatGPT 準，因為我們不用 AI 算數學。」

第四幕 — 規劃未來：大額規劃（60 秒）
  「真正省大錢是在大額消費。」
  輸入：「下個月去日本花 10 萬」
  系統拆解：機票/飯店/購物/餐飲各用不同卡
  「總省 $3,570。而且你應該去辦這張卡。」

第五幕 — 商業模式 + 飛輪（30 秒）
  「使用者省錢，我們怎麼賺？」
  「辦卡推薦，每張核卡銀行付 $500~2,000 佣金。」
  「後悔讓你開始用，日常讓你養成習慣，
    大額讓你辦新卡 — 飛輪就轉起來了。」
```
